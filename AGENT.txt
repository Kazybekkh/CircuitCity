You are a coding agent working on the `canvas` branch of the `circuits-as-a-quest` monorepo.

## Your mission
Build an infinite canvas circuit editor — a Tinkercad-style drag-and-drop builder where users can construct circuits from scratch. As the circuit is built, it continuously triggers the sim engine and updates the RPG story in real time. The circuit editor and the RPG view live side by side.

## Stack
- Next.js 14 (App Router)
- React Flow 11 (infinite canvas, custom nodes)
- Zustand (shared with frontend branch via same store shape)
- Tailwind CSS

## Step 2 — Component palette
A vertical tray of draggable circuit components:
- Resistor, Capacitor, Inductor, LED, Diode, Voltage Source, Current Source, Switch (open/closed toggle), Ground, Wire node
- Each palette item is a 48x48 pixel art icon matching the RPG landmark it maps to (resistor = swamp icon, LED = bonfire icon, etc.)
- Drag from palette → drop onto canvas → spawns a React Flow node
- Show component value input inline on the node (e.g. "1kΩ", "10µF") — editable double-click

## Step 3 — Infinite canvas (React Flow)
Custom React Flow node types for each ComponentType:

Each node renders as:
- A pixel-art schematic symbol (standard IEEE symbols, drawn in SVG inside the node)
- Below it: the component value in monospace text
- Colour-coded border matching its RPG biome colour
- On select: shows a right-click context menu with Delete and Edit Value

Wire connections:
- Standard React Flow edges with custom style: 2px solid, colour = current magnitude (blue=low, yellow=mid, red=high) — colour updates after each simulation
- Connection handles on all four sides of each node

Canvas features:
- Infinite pan and zoom (React Flow built-in)
- Grid snap (16px grid — pixel art aesthetic)
- Multi-select with shift-click and drag-select
- Delete selected with Backspace/Delete key
- Undo/redo with Ctrl+Z / Ctrl+Y (maintain history stack in Zustand)
- Export canvas as CircuitGraph JSON (download button)

## Step 4 — CircuitGraph derivation
Implement `canvasToCircuitGraph()` that converts React Flow node/edge state to a valid CircuitGraph:
- Map each React Flow node to a CircuitNode (id, type, value from label, position)
- Map each React Flow edge to a CircuitEdge (id, from = source node id, to = target node id)
- Validate: every circuit must have at least one voltage source and one ground — show inline warning if not
- Handle floating nodes (nodes with no edges) — mark them as open circuit

## Step 5 — Real-time story updates
This is the key feature: as the user builds, the RPG story updates itself.

Debounce strategy:
- On any canvas change (node added/moved/edited, edge connected/deleted) set isDirty=true
- After 800ms of no further changes, automatically:
  1. Call canvasToCircuitGraph()
  2. POST to /api/simulate with the derived graph
  3. If simulation succeeds, update sceneConfig in the Zustand store
  4. The QuestView (frontend branch) reacts to the store change and re-renders the RPG scene
  5. POST to /api/narrate with the new sceneConfig
  6. ExplanationPanel updates with the new narrative and audio

Show a live status indicator in the canvas toolbar:
- "Drawing..." (user is placing components)
- "Simulating..." (waiting for /api/simulate)
- "Quest updated!" (flash green, 1.5s)
- "Circuit incomplete" (missing source or ground)
- "Fault detected" (sim returned a fault — show which component in red)

## Step 6 — Side-by-side layout
The canvas branch adds a new route: `/build`

Layout at /build:
- Left 55%: SchematicBuilder (infinite canvas)
- Right 45%: QuestView + ExplanationPanel (imported from frontend branch components)
- The two panels are live-synced via Zustand — building on the left instantly affects the RPG on the right
- Toolbar at the top: component palette, zoom controls, undo/redo, simulate button (manual trigger), save to MongoDB button

## Constraints
- The canvas must never block the UI thread — debounce all simulation calls
- React Flow node custom types must be registered before render — no dynamic type registration
- Wire colours must update after each simulation result — map edge current to colour in the store update
- All pixel art icons must be SVG (not raster images) — drawn as clean schematic symbols
- The entire /build route must work without the sim backend running — show a "Sim offline" badge and disable auto-simulate, but canvas editing still works
